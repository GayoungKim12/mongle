# CLAUDE.md — mongle 프로젝트 기반 컨텍스트

> 이 파일은 Claude Code가 작업 시 자동으로 참조하는 프로젝트 규칙 및 배경지식입니다.
> 새 작업을 시작할 때 항상 이 파일을 기준으로 판단하세요.

---

## 1. 프로젝트 개요

**mongle**은 사용자가 감정을 자유롭게 글로 쓰면, AI가 이를 구조화된 위로와 콘텐츠(에세이·음악·영상)로 변환해 돌려주는 **모바일 퍼스트 AI 감정 일기 앱**입니다.

> 한 줄 요약: "생각을 구름으로 흘려보내면, 위로가 되어 돌아온다."

핵심 감정 경험:
- 글쓰기 = "생각을 내보내는 행위"처럼 느껴져야 함
- 결과 화면 = "따뜻한 위로를 받는 느낌"이어야 함
- UI는 절대 압도적이거나 자극적이어서는 안 됨

## 3. 디자인 시스템

### 색상 팔레트
```
--color-pink:       #FADADD   /* 기본 배경/버튼 시작 */
--color-purple-light: #E6D6FF /* 보조 배경 */
--color-purple:     #CBB8FF   /* 강조 / 선택 상태 */
--color-blue-light: #DFF4FF   /* 액센트 */
--color-white-glass: rgba(255,255,255,0.6) /* 글래스모피즘 카드 */
```

### 타이포그래피
- 폰트: Pretendard (또는 동일 계열 가독성 우선 폰트)
- Title: `font-weight: 600`
- Body: `font-weight: 400`
- 장식체, 손글씨체 사용 금지
- 감정 전달은 **폰트 스타일이 아닌 색상과 여백**으로만

### 형태 & 질감
- 모든 카드 radius: `20px ~ 28px`
- CTA 버튼: pill shape, pink→purple 그라디언트
- 배경: 워터컬러 텍스처 + 미세 paper grain/noise overlay
- 강한 그림자 금지 → `box-shadow: 0 4px 20px rgba(0,0,0,0.06)` 수준
- 글래스모피즘: `backdrop-filter: blur(12px)`, 반투명 흰 배경

## 4. 화면 구성 (Pages)

### Screen 1 — 감정 작성 페이지 (`/write`)

| 영역 | 내용 |
|------|------|
| 상단 | 날짜 (small, subtle) + 프롬프트: "오늘은 어떤 하루였나요?" |
| 중앙 | 텍스트 입력 영역 (border 없음, 글래스모피즘) |
| Placeholder | "지금 머릿속 생각을 편하게 적어보세요…" |
| 하단 | CTA 버튼: "☁️ 구름 만들기" |

UX 규칙:
- 타이핑 중 배경이 미세하게 변화 (색상 shift)
- 배경에 숨쉬는 듯한 느린 모션 유지
- 입력창에 포커스 시 부드러운 glow 효과

### Screen 2 — 감정 결과 페이지 (`/result`)

렌더링 순서 (순차 fade-in):
1. 공감 캐릭터 셀렉터
2. AI 공감 텍스트
3. 에세이 카드
4. 음악 추천 (2개 고정)
5. 영상 추천 (2개 고정)

#### 공감 캐릭터 셀렉터
- 옵션: 👶 어린이 / 👩 엄마 / 👨 아빠
- UI: rounded segmented control
- 선택 상태: 그라디언트 배경 + 흰 텍스트
- 선택 변경 시 AI 공감 텍스트 톤 변경

공감 톤 가이드:
- 어린이: 단순하고 다정한 말투
- 엄마: 따뜻하고 보살피는 말투
- 아빠: 차분하고 든든한 말투

#### AI 공감 텍스트
- 카드 없이 플로팅 텍스트로 표시
- 애니메이션: blur → clear fade-in

#### 에세이 카드
- 워터컬러 일러스트 배경 + 반투명 오버레이
- radius: `28px`
- 텍스트 3~5줄 최대 (엽서/책 페이지 느낌)

#### 음악 카드 (2개)
- 섹션 타이틀: "마음을 정리해줄 음악"
- 반투명 흰 배경, radius `20px`, 미니멀 shadow
- 곡 제목 + 추천 이유 한 줄

#### 영상 카드 (2개)
- 섹션 타이틀: "잠깐 쉬어갈 영상"
- 음악 카드보다 큰 썸네일
- radius `24px`
- tap 시 살짝 올라오는 lift 효과

## 5. 모션 & 인터랙션 원칙

- 트랜지션: 항상 soft fade-in (급격한 전환 금지)
- 결과 페이지 요소: 순차적으로 등장 (동시 렌더 금지)
- 애니메이션 속도: 느리고 차분하게 (easing: `ease-out`)
- blur→clear 효과를 적극 활용
- 버튼 탭 시: scale down 살짝 (press 피드백)

|------------|---------|

## 7. 금지 사항 (절대 하지 말 것)

- 강한 그림자 (`box-shadow` 진하게)
- 고대비 색상 조합
- UI 과부하 (정보/버튼 너무 많음)
- 게임화 요소 (포인트, 뱃지, 레벨 등)
- 자극적이거나 화려한 애니메이션
- 에러 상태의 붉은 색상 → 대신 부드러운 메시지로 처리
- 추천 콘텐츠 개수 초과

---

## 7-1. 핵심 UX 인터랙션 (추가 확정)

| 인터랙션 | 동작 |
|---------|------|
| 입력 중 배경 | 부드럽게 색감이 흐름 (debounce 처리) |
| 저장/제출 시 | 텍스트가 구름 형태로 변환되는 애니메이션 |
| 에세이 카드 등장 | blur → clear 순차 애니메이션 |
| 전체 경험 | "흐르고 번지는 느낌" — 급격한 전환 없음 |
| 영상 카드 hover | 부드럽게 위로 떠오름 (lift effect) |

---

## 7-2. AI 처리 흐름

```
사용자 텍스트 입력
  → AI 분석 (감정 파악)
  → 구조화 (에세이 생성)
  → 공감 메시지 생성 (선택 캐릭터 톤)
  → 콘텐츠 큐레이션 (음악 2 + 영상 2)
  → 회복 경험 제공
```

- AI 응답은 **스트리밍 UI**로 표시 (글자가 흘러들어오는 효과)
- 각 섹션은 스트리밍 완료 후 순차적으로 나타남
- 로딩 상태: 스피너 대신 부드러운 shimmer / breathing 애니메이션

---

## 8. AI 응답 프롬프트 원칙

Claude API 호출 시 시스템 프롬프트 기본 톤:
- 판단하지 않고 공감한다
- 짧고 따뜻하게
- 선택된 공감 캐릭터 톤에 맞춤
- 해결책 제시보다 감정 인정 우선
